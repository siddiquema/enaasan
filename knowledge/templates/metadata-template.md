# Metadata Template

Copy this block to the top of any new publication's Markdown source. Fill in every field — see `architecture/metadata-standard.md` for what each one means and its allowed values.

```yaml
---
document_id: ENA-[TYPE]-[AUDIENCE]-[slug]-v0.1
title: "[Full reader-facing title]"
subtitle: "[Optional — delete this line entirely if unused]"
version: 0.1
status: Draft
author: "[Name or role]"
audience: "[Graduates | Schools (10th) | Schools (12th) | Parents | Careers | Workplace | All]"
reading_level: "[Plain-language description]"
estimated_reading_time: "[e.g. 8 minutes — delete if using estimated_completion_time instead]"
estimated_completion_time: "[e.g. 2 hours across 1 week — delete if using estimated_reading_time instead]"
difficulty: "[Not applicable | Beginner | Intermediate | Advanced]"
tags: [tag-one, tag-two]
prerequisites: ["None"]
related_documents: []
related_careers: []
related_skills: []
last_reviewed: "[YYYY-MM-DD]"
review_due: "[YYYY-MM-DD]"
license: "Free to read, download, and share for personal and educational use. Not for resale or republishing without permission. © enaasan.online"
---
```

Do not leave any bracketed placeholder unfilled in a document that has moved past `Draft` status.
