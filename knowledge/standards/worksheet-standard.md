# Worksheet Standard

Rules for the fill-in-the-blank worksheet portions used inside Workbooks (see `templates/workbook-template.md`) — distinct from a whole Workbook's standard (there isn't a separate `workbook-standard.md`; a Workbook's overall completeness is covered by `publication-standard.md` plus this file for its worksheet components specifically).

## What makes a worksheet prompt well-formed

- [ ] The prompt is a specific, answerable question or instruction — not a vague topic label. Compare "My real constraints — money, location, family expectations, time:" (well-formed) against "Think about your constraints" (too vague to act on).
- [ ] Each prompt can genuinely be answered in the space/time implied — don't ask a question that needs paragraphs of reflection but give it one ruled line.
- [ ] Prompts build on each other in a logical order within a section, per the sequencing principle in `learning-path-standard.md`.

## Formatting

- Use the fill-in-line pattern from `publishing/markdown-standard.md` (a labelled prompt followed by a blank line) consistently — don't mix in other input styles (dropdowns, multi-part grids) without a clear reason tied to the content's actual shape (a genuinely tabular prompt, like a skill-gap tracker, correctly uses a table instead — see the existing 90-Day Plan's Weeks 4–6 skill-gap table as the reference case).
- Checkboxes (`- [ ]`) are used for a concrete, checkable end-of-section outcome, not for open-ended reflection prompts — those get a fill-in line instead.

## Honesty in framing

- Worksheets that ask a reader to be candid about themselves (constraints, weaknesses, uncertainty) should explicitly reassure that the answers are private and there's no "impressive" answer expected — see the existing 90-Day Plan's Week 1 framing ("Nobody sees this page but you — so there is no point being impressive here. Be accurate instead.") as the reference tone.

## Print consideration

- Ruled fill-in lines must render with enough vertical space to actually be written on by hand when printed — this is a functional requirement, not just a visual one. A worksheet that looks fine on-screen but leaves no real writing room when printed has failed its purpose.
