# publications/parents/

## Purpose

Real publications aimed at parents, guardians, teachers, and counsellors — supporting a young person's decision without imposing on it. Corresponds to the `PARENT` audience code in `architecture/document-id-standard.md`, and to the `/parents` topic hub concept in `docs/website-update-spec.md` §10 (overlapping with the existing "dedicated parent guidance section" request tracked as GitHub issue KM-04).

## Contents

Empty today. Each future publication gets its own subfolder, named by slug.

## Naming Convention

`publications/parents/<slug>/source.md`.

## Rules

Same as the parent `publications/README.md`. Tone for this audience follows `branding/tone-of-voice.md`'s specific guidance: measured pace, parental concern acknowledged as legitimate before being addressed.

## Examples

- A future Guide formalising the existing "myths worth reconsidering" content on `guide.html`, and the FAQ's existing "For Parents" answers (Q31–Q32), into a fuller standalone document.

## Future Expansion

- Likely early content: a Playbook for common parent-child disagreement scenarios (a child wanting a non-traditional path, disagreement over drop years), Knowledge Cards explaining terms parents encounter but may not know (lateral entry, CUET, apprenticeship).
